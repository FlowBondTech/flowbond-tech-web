## Project Overview

This is a React-based landing page for FlowBond/DANZ - a "dance-to-earn" cryptocurrency platform. The application features a modern neon aesthetic with purple/pink theming, advanced visual effects, and a component-based architecture built with Vite and React Router.

## Development Commands

```bash
# Development server (runs on default Vite port)
npm run dev

# Production build
npm run build

# Preview production build
npm run preview

# Quick server startup (using provided script)
./run-server.sh
```

## Architecture & Structure

### Core Architecture
- **Framework**: React 18 with Vite build system
- **Routing**: React Router DOM v7 with BrowserRouter (configurable to HashRouter for GitHub Pages)
- **Styling**: Custom CSS with CSS variables and modern features (Grid, Flexbox, animations)
- **Effects System**: Custom hooks-based neon effects with particle systems and animations

### Key Architectural Patterns

**Layout System**:
- `Layout.jsx` wraps all routes and initializes visual effects
- Route-based opacity transitions and animation re-triggering on navigation
- Particles container rendered at layout level for consistent background

**Effects Architecture**:
- Modular hook system in `useNeonEffects.js` with separate concerns:
  - `useParticles()` - Dynamic particle background generation
  - `useScrollIndicator()` - Progress bar with gradient styling
  - `useMagneticButtons()` - Mouse-following button interactions
  - `useGlowEffects()` - Hover-based glow overlays
  - `useTypingEffect()` - Animated text typing for hero section
  - `useCounters()` - Intersection observer-based number animations
  - `useRevealAnimations()` - Scroll-triggered fade-in effects

**Component Structure**:
- **Multi-route Setup**: Homepage (Hero + sections), `/danz` page, `/tokenomics` page
- **Reusable Components**: Shared between routes with consistent styling
- **Effect Integration**: Components use CSS classes (`.animate`, `.btn`, etc.) that hook system targets

### Styling System

**CSS Architecture**:
- CSS custom properties for theming in `:root`
- Neon color palette: `--neon-pink` (#ff6ec7), `--neon-purple` (#b967ff)
- Responsive design with mobile-first approach
- Animation keyframes dynamically generated by JavaScript hooks

**Visual Effects**:
- Particle system with randomized floating animations
- Magnetic button interactions with transform-based movement
- Gradient-based glows and neon styling throughout
- Smooth reveal animations using Intersection Observer API

## Deployment Configuration

### Multi-Platform Support
- **Netlify**: Configured with `public/netlify.toml` for SPA routing
- **Vercel**: Configured with `vercel.json` for catch-all rewrites
- **GitHub Pages**: Router can be switched to HashRouter (see App.jsx comment)

### Routing Configuration
All platforms redirect unknown routes to `/index.html` for client-side routing compatibility.

## Key Technical Considerations

### Performance
- Effects system uses cleanup functions to prevent memory leaks
- Intersection Observer for scroll-triggered animations (better than scroll listeners)
- Particle animations use CSS transforms for hardware acceleration
- Route transitions include opacity management to prevent flashing

### Browser Compatibility
- Modern CSS features (CSS Grid, custom properties, transforms)
- ES6+ JavaScript (modules, arrow functions, destructuring)
- Requires browsers supporting Intersection Observer API

### Effect System Dependencies
- Effects rely on specific CSS classes being present in components
- DOM manipulation occurs after component mount with setTimeout delays
- Route changes trigger effect re-initialization through Layout component

## Component Development Patterns

### Adding New Components
- Use `.animate` class for reveal animations
- Use `.btn` class for magnetic button effects
- Follow existing CSS variable naming for consistent theming
- Include proper cleanup in effects that modify DOM

### Effect Integration
- Effects initialize on component mount through Layout wrapper
- CSS classes serve as selectors for effect targeting
- Route changes automatically re-trigger animations
- Particle background persists across all routes

## Dependencies & Technical Stack
- **React**: 18.2.0 with hooks-based architecture
- **React Router DOM**: 7.7.1 for client-side routing
- **Framer Motion**: 12.23.9 (available but not currently used)
- **Vite**: 7.0.6 for build tooling and dev server
- **No external CSS frameworks** - custom CSS implementation
